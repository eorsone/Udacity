CSS Syntax

In CSS there are two rulesets that are in play, Seletor and Declaration Block. Selectors determine which element will the the ruleset apply to; while, Declaration Blocks determines how the ruleset will modify the element(the rules that the element will follow).

example

  • li { color: green; }
In the example above, the li is the selector, which states that CSS will apply to all p elements, while the declaration block is the writing within curly braces, color: green. This is the format that CSS rules are writen in. Selectors are followed by the declaration block.

example

  • p { width: 50%; }
    .weird { color: purple; font-size: large; }
.weird, being the selector, affects all elements with the .weird class. In the declaration block, the descriptive name(width, color, font-size, etc) is known as the property, after the colon(:), the value is written followed by a semicolon(;). In the example above, the width is the property and the value is 50%.

There are a multitude of selectors that can be used in CSS. Here are few different types that you will see.

  • Typ e Selector - names the type of the html element and apply style to all.. Some examples are p, em, h2, body, etc
  • Class- A group of things with the same characteristics. To create a class, add class="" to the html element to catagories that element. When applying the class into CSS, you'll write the selector with a "." followed by the class name.
  • Id - Similar to class but only applies to one element. Class is written in CSS with a "#" in front
  • Descendant Selectors - Allows you to specifically target an element nested in another element. To apply, write the parent element's name first followed by the child element's name.
  • For more on selectors, visit Mozilla

Cascading and Specificity

Cascading has to do with how the DOM tree gets its style define, specifically, it will style a node and all of its descendence. If there are two sets of code attempting to affect the same property, the one which is most specific will be applied.

Units

In CSS, the basic unit used is pixels(px). A pixel is 1/96 of an inch or 1 inch is equal to 96 pixels. Another unit is the point(pt.) A point is 1/72 of an inch or 72 point = 1 inch. Pixels and points are based on inches while some other units are based on something on the page. Em for example is based on the font size of the text in the element.

Boxes

In previous lessons on html, we discovered that some elements such as <p> and <div> put a an invisible box around the content of the element. CSS allows us to customize the width and height of the box as well as the following:

  • Padding - space around the content of the box(between the content and the border)
  • Border - area around the box
  • Margin - the space that the browser makes between this box and other boxes.
You can customize these properties by using px or ems. If you want to move the boxes toward the right rather than down, you can use the command float: right. This command tells the browser to try and move the box toward any direction of the page, it is commonly used to move boxes around.

Percentages

When defining sizes in CSS, it is more common to use a percentage instead of an exact amount of px or em. Percentages go by the width or height of the box that it is inside. For example, if a <div class = box> is set to 100px and we set another div inside it to 50%, the div inside is going to be 50px. If the box is at the top of the document, 50% means 50% of the whole page. One thing to keep in mind, by default, width doesn't include the border and never includes the padding. This means that if you want two boxes to fit next to each other using the float command, you'd want to set the percentage below 50%. Percentages can also be used for scaling the size of elements up or down.

Applying style to documents

There are two common ways to apply CSS to a document. One way is to directly apply the style under the style tab at the top of the html;however, this can get a bit messy when using a lot of CSS and applying it to multiple documents(it isn't as efficient.) Another way is to put the CSS into a seperate file and then reference the file in the head of the html. To do this, apply the special tag <link rel="stylesheet" href = "(url of stylesheet file)">(If file is in same directory, you can just type the file name). Keep in mind that you can only use one method at a time. A stylesheet and a style element cannot be in the same doc.
